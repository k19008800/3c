# 多角色业务流程验证

**验证目标**: 以用户使用角色产生各类型的业务数据，来推演整个项目所需要的所有功能，验证项目代码正确性。

**验证方法**:
1. 模拟不同角色用户执行完整业务流程
2. 使用浏览器实际渲染为标准，不只看HTTP状态码
3. snapshot验证页面内容和交互
4. 发现问题立即修复

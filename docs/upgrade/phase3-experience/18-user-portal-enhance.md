# 18 — 用户端体验增强

> **后端**: 0.5 人天 | **前端**: 1.5 人天 | **依赖**: 02 一键复制、08 时长配额、17 术语系统

---

## 1. 背景与目标

**问题**：用户端页面功能分散，缺少一站式的"使用指南"体验。新用户不知道如何开始、不知道去哪里看消耗、没有集成的一键接入引导。

**目标**：用户端仪表盘升级为个人工作台、文档页面增强为 SDK 中心、消息通知实时推送。

## 4. 通知中心增强

### 现状
已有通知页面（`pages/Notifications.tsx`），但只是被动展示。

### 升级

```typescript
// WebSocket 实时推送
// 连接方式：页面加载时通过 WebSocket 或 Server-Sent Events

const notificationWS = new WebSocket(
  `${location.protocol === 'https:' ? 'wss:' : 'ws:'}//${location.host}/api/v1/user/notifications/ws`
)

notificationWS.onmessage = (event) => {
  const notification = JSON.parse(event.data)
  
  // Toast 弹出
  toast({
    title: notification.title,
    description: notification.message,
    icon: notification.type === 'alert' ? '🔴' : 'ℹ️',
    duration: 5000,
    action: notification.link ? {
      label: '查看',
      onClick: () => window.location.href = notification.link,
    } : undefined,
  })
  
  // 更新侧边栏通知角标
  updateBadge(notification.unreadCount)
}
```

### 通知类型

| 通知类型 | 触发条件 | 示例消息 |
|----------|---------|---------|
| `quota_alert` | 配额使用超过 80% | "月度 Token 配额已使用 85%，剩余 15% 将在月底重置" |
| `balance_low` | 余额低于阈值 | "余额仅剩 ¥5.00，请及时充值避免服务中断" |
| `anomaly` | 异常检测 | "检测到 API Key 'prod-key' 在 5 分钟内失败 10 次，请检查" |
| `announcement` | 管理员发布公告 | "系统将于 07-20 02:00-04:00 进行升级维护" |
| `recharge` | 充值成功/审核通过 | "充值 ¥100.00 已到账，当前余额 ¥1,234.56" |

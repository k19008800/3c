### 发现的问题 / 备注

1. **API Key 删除 dialog 处理问题**: `browser` 工具的 `dialog(accept=true)` 操作在处理浏览器原生 confirm 对话框时存在超时问题（TimeoutError），无法完成确认。这可能是因为 dialog 触发了 DOM 模态覆盖层或 Playwright 的 dialog 监听未能及时捕获。建议：
   - 考虑将原生 `window.confirm` 替换为自定义模态组件（如 Ant Design Modal.confirm），便于自动化测试
   - 或确认 Playwright 版本与 Chrome 版本兼容性

2. **PortInUseError 间歇出现**: 测试过程中偶发 `PortInUseError: Port 18800 is already in use` 错误，重试后通常恢复。可能与浏览器自动重启有关（测试期间 PID 从 23540 变为 6968）。

3. **导航到 SPA 路由直接返回 404**: `navigate({url: ".../api-keys"})` 直接返回了 `{"message":"Route GET:/api-keys not found"}` JSON（因为 dev server 的 SPA 回退未正确处理）。使用侧边栏点击跳转一切正常。

## 🔴 发现的问题

### 严重问题

1. **邮件模板路由缺失**
   - **表现**: 前端"邮件模板"菜单项无法加载数据 (404)
   - **原因**: 后端无 `email-templates` 相关路由
   - **影响**: 管理员无法在后台编辑 4 套邮件模板（注册验证/密码重置/充值确认/实名结果）
   - **修复**: 需在 `admin/system.ts` 中添加 CRUD 路由

2. **Fastify JSON Body BOM 解析 Bug**
   - **表现**: 带 UTF-8 BOM 的 JSON body 被解析为 `null`，导致 `null.captchaSession` 500 错误
   - **原因**: `app.ts` 中 BOM stripping 代码未正确生效
   - **影响**: 特定客户端工具发送的含 BOM 请求会 500
   - **修复**: 调试 `app.ts` 第 25-27 行 BOM 处理逻辑

### 次要问题

3. **路由路径文档不一致**
   - API Keys: 文档 `/api/v1/keys` → 实际 `/api/v1/api-keys`
   - 实名审核: 文档 `/api/v1/admin/reviews` → 实际 `/api/v1/admin/real-name-reviews`
   - Dashboard: 文档 `/api/v1/admin/dashboard` → 实际 `/api/v1/admin/dashboard/stats`
   - 系统配置: 文档 `PUT /api/v1/admin/system/configs` → 实际 `PATCH /api/v1/admin/configs/:key`

4. **角色字段歧义**
   - `users.role` 和 `admin_roles` 表角色不一致
   - admin@3cloud.ai 在用户列表显示 role=agent，登录返回 role=super_admin

5. **测试用户密码不统一**
   - 非管理员用户无公开密码，无法完成登录测试

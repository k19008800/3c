# 3cloud 角色推演与功能验证计划

## 一、业务模型分析

### 1.1 AI Token 聚合平台的核心价值主张
```
上游供应商（DeepSeek/OpenAI/Claude等） → 3cloud平台（聚合+路由+计费） → 下游用户（开发者/企业/代理商）
```

### 1.2 关键成功因素
1. **供应链稳定性**：多供应商冗余，智能路由
2. **计费准确性**：精确到Token的实时计费
3. **用户体验**：简单易用的API和清晰的账单
4. **运营效率**：自动化管理和风险控制
5. **财务透明**：清晰的成本收益分析

## 二、多角色功能需求推演

### 2.1 终端用户（API消费者）

**典型用户画像**：
- 独立开发者：调用量小，关注价格和易用性
- 中小企业：中等调用量，需要稳定性和技术支持
- 大型企业：高调用量，需要定制化方案和合同

**核心业务流程**：
```
注册/登录 → 查看模型价格 → 创建API Key → 充值余额 → 调用API → 查看账单 → 申请发票
```

**所需功能验证**：
1. ✅ 用户注册和登录
2. ✅ API Key 管理（创建/查看/禁用）
3. ✅ 余额查询和充值（多种支付方式）
4. ✅ 调用日志查看（实时和历史）
5. ✅ 用量统计和分析
6. ✅ 发票申请和管理
7. ✅ 个人账户安全设置
8. ✅ 实名认证流程

### 2.2 代理商（渠道合作伙伴）

**典型画像**：
- 技术集成商：为客户提供AI能力集成
- 行业解决方案商：垂直行业AI应用
- 流量平台：自带用户和流量

**核心业务流程**：
```
申请成为代理商 → 获取推广码 → 发展下级用户 → 佣金结算 → 提现申请 → 业绩分析
```

**所需功能验证**：
1. ✅ 代理商入驻审核流程
2. ✅ 推广码生成和管理
3. ✅ 下级用户管理和统计

5. ✅ 提现申请和审核流程
6. ✅ 代理商数据面板
7. ✅ 结算对账功能

## 三、端到端业务流程验证

### 3.1 用户完整生命周期验证

**场景1：新用户从注册到消费**
```
1. 用户注册账户
2. 完成实名认证
3. 创建API Key
4. 充值100元
5. 调用API消耗20元
6. 查看调用日志
7. 申请发票
8. 查看余额和账单
```

**验证重点**：
- 流程是否顺畅无阻
- 费用计算是否准确
- 数据是否实时同步
- 用户体验是否友好

### 3.2 代理商业务流程验证

**场景2：代理商发展下级用户**
```
1. 申请成为代理商（审核通过）
2. 生成推广码
3. 下级用户通过推广码注册
4. 下级用户消费1000元
5. 查看佣金明细（假设10%佣金）
6. 申请提现100元
7. 查看提现审核状态
```

**验证重点**：
- 佣金计算准确性
- 上下级关系正确性
- 提现审核流程完整性
- 数据隔离安全性

### 3.3 财务对账流程验证

**场景3：月末财务结算**
```
1. 生成月度结算单
2. 导出代理商佣金明细
3. 导出供应商应付账单
4. 核对平台总余额
5. 生成财务报表
6. 处理企业发票申请
```

**验证重点**：
- 数据一致性（各表关联正确）
- 金额计算准确性
- 导出格式完整性
- 审计痕迹完整性

## 四、技术架构验证

### 4.1 核心服务验证
1. **API路由服务**：多供应商智能路由和降级
2. **计费引擎**：实时Token级计费准确性
3. **限流服务**：多维度限流策略有效性
4. **安全服务**：认证、授权、审计完整性

### 4.2 数据一致性验证
1. **余额一致性**：users.balance vs balance_logs
2. **佣金一致性**：agents.commission_balance vs commission_logs
3. **调用统计一致性**：call_logs统计 vs 实时Redis计数
4. **财务一致性**：充值+消费+提现=余额变化

### 4.3 性能边界验证
1. **高并发调用**：模拟1000+ QPS调用
2. **大数据量统计**：百万级call_logs查询性能
3. **实时监控**：仪表盘数据实时性
4. **导出功能**：大数据量CSV导出性能

## 六、风险点和应对策略

1. **余额负数问题**：超级管理员显示-10元余额
